Modèle de Concurrence

AI Trading Firm - Exécution Parallèle des Agents

asyncio Fan-out / Fan-in Barrier Sync Fault Tolerant

1. Vue d'Ensemble

Le système utilise un modèle de concurrence en 3 phases pour respecter la constitution CLAUDE.md :

SIGNAL AGENTS
Parallèle
→
BARRIER
Sync
→
CIO
Décision
→
RISK
Validation
→
EXEC
Ordre
1
FAN-OUT : Exécution Parallèle
Tous les Signal Agents démarrent simultanément via asyncio.gather(). Chaque agent a un timeout individuel de 5 secondes. Les agents s'exécutent vraiment en parallèle (concurrent I/O).
2
BARRIER : Synchronisation
La SignalBarrier collecte tous les signaux. Timeout global : 10 secondes. Si timeout : le CIO décide avec les signaux disponibles (signaux partiels). Les agents lents ou défaillants sont exclus du cycle.
3
SÉQUENTIEL : Pipeline Strict
Pas de parallélisme - Ordre garanti : CIO → Risk → Execution. Chaque étape await la précédente. Un rejet à n'importe quelle étape stoppe le cycle.

3. Implémentation: SignalBarrier

La barrière de synchronisation collecte les signaux et gère les timeouts :

@dataclass class SignalBarrier: """ Barrière de synchronisation pour les signaux. Implémente le pattern fan-in avec timeout. """ expected_agents: set[str] # Agents attendus timeout_seconds: float # Timeout max (10s) signals: dict[str, SignalEvent] = field(default_factory=dict) _event: asyncio.Event = field(default_factory=asyncio.Event) def add_signal(self, agent_name: str, signal: SignalEvent) -> bool: """Ajoute un signal. Retourne True si COMPLET.""" self.signals[agent_name] = signal if self.is_complete(): self._event.set() # Libère la barrière return True return False def is_complete(self) -> bool: """Vérifie si tous les agents ont reporté.""" return set(self.signals.keys()) >= self.expected_agents async def wait(self) -> dict[str, SignalEvent]: """ Attend tous les signaux OU timeout. Retourne les signaux disponibles (complets ou partiels). """ try: await asyncio.wait_for( self._event.wait(), timeout=self.timeout_seconds ) logger.info(f"Barrier complete: {len(self.signals)} signals") except asyncio.TimeoutError: missing = self.expected_agents - set(self.signals.keys()) logger.warning( f"Barrier TIMEOUT. Missing: {missing}" ) return self.signals # Retourne ce qu'on a

4. Implémentation: Exécution Parallèle

async def process_market_event(self, market_data: MarketDataEvent) -> None: """ Traite un événement de marché. PHASE 1: Fan-out parallèle PHASE 2: Barrier synchronisation PHASE 3: Séquence CIO → Risk → Execution """ # ═══════════════════════════════════════════════════════════════ # PHASE 1: FAN-OUT - Exécution PARALLÈLE des Signal Agents # ═══════════════════════════════════════════════════════════════ # Créer une nouvelle barrière pour ce cycle barrier = SignalBarrier( expected_agents={agent.name for agent in self._signal_agents}, timeout_seconds=10.0, ) # Lancer TOUS les agents en parallèle signal_tasks = [ self._run_signal_agent_with_timeout(agent, market_data, barrier) for agent in self._signal_agents ] # Exécution VRAIMENT parallèle (concurrent) await asyncio.gather(*signal_tasks, return_exceptions=True) # ═══════════════════════════════════════════════════════════════ # PHASE 2: BARRIER - Synchronisation (Fan-in) # ═══════════════════════════════════════════════════════════════ signals = await barrier.wait() if not signals: logger.warning("No signals received, skipping cycle") return # ═══════════════════════════════════════════════════════════════ # PHASE 3: SÉQUENTIEL STRICT - CIO → Risk → Execution # ═══════════════════════════════════════════════════════════════ # Étape 3.1: CIO décide (UNIQUE décideur) decision = await self._cio_agent.make_decision(signals) if decision is None: return # Pas d'action # Étape 3.2: Risk/Compliance valide (SÉQUENTIEL après CIO) validation = await self._risk_agent.validate(decision) if not validation.approved: return # Rejeté # Étape 3.3: Execution exécute (SÉQUENTIEL après Risk) await self._execution_agent.execute(decision, validation)

5. Gestion des Timeouts et Agents Défaillants

async def _run_signal_agent_with_timeout( self, agent: SignalAgent, market_data: MarketDataEvent, barrier: SignalBarrier, ) -> None: """ Exécute un signal agent avec timeout individuel. 3 cas possibles: 1. Succès → Signal ajouté à la barrière 2. Timeout → Agent ignoré, log d'erreur 3. Exception → Agent ignoré, log d'erreur """ try: # Timeout INDIVIDUEL par agent (5s) signal = await asyncio.wait_for( agent.generate_signal(market_data), timeout=5.0, ) if signal: barrier.add_signal(agent.name, signal) logger.debug(f"{agent.name}: Signal OK") else: # Pas de signal (normal) → signal FLAT barrier.add_signal(agent.name, FlatSignal(agent.name)) except asyncio.TimeoutError: # CAS 2: Agent LENT - on continue sans lui logger.error(f"{agent.name}: TIMEOUT after 5s") self._stats[agent.name]["timeouts"] += 1 # NE PAS ajouter à la barrière except Exception as e: # CAS 3: Agent en ERREUR - on continue sans lui logger.error(f"{agent.name}: ERROR - {e}") self._stats[agent.name]["errors"] += 1 # NE PAS ajouter à la barrière
Important: Les agents en timeout ou erreur ne sont PAS ajoutés à la barrière. Le CIO prendra sa décision avec les signaux disponibles uniquement.
